# Test gate (audit P2) — pre-push money smoke on an isolated Neon branch

## Why

The audit's one confirmed-medium structural gap: `npm test` is invoked by nothing; push = deploy.
Since push IS the deploy trigger, only a pre-push hook can actually BLOCK a bad deploy; a GitHub
Action can only alarm after the fact. Decided shape (Dallas 2026-07-13): pre-push money-path smoke
first, Action tripwire second (stage 2, separate follow-up once stage 1 proves out).

## Design (stage 1)

**Isolated DB:** persistent Neon branch `ci-smoke` (`br-twilight-surf-adf8eghx`, parent =
production), PROVISIONED 2026-07-13. Per run: reset-to-parent via Neon REST API → fetch connection
URI → run `initDb` against it (this also validates any schema.sql change in the push BEFORE prod
boot replays it — a gap nothing else covers) → run the smoke suite list serially with
`DATABASE_URL` overridden (dotenv does not override existing env). Suites self-fixture; the
prod-parented copy supplies schema + seed data exactly like the shared dev DB does today.

**Trigger:** `.husky/pre-push` runs the smoke only when the push range touches `server/`
(mirrors the existing `client/`-only CI-build gate). Docs-only pushes stay instant.

**Credential:** `NEON_API_KEY`, read from env or `~/.secrets/neon_api_key`. UNTIL the key exists
the hook prints a loud red "money smoke SKIPPED — not yet blocking" banner and allows the push
(fail-open so pushes are not bricked before setup); once the key is present the gate is HARD
(fail = push blocked; `git push --no-verify` stays the documented emergency escape, same as the
client build gate). **Dallas action item: mint a Neon API key** (console.neon.tech → account →
API keys) and drop it in `~/.secrets/neon_api_key`.

**Smoke list:** `scripts/money-smoke-list.txt`, one repo-relative test file per line, assembled
from EXISTING money-path suites (payroll accrual/clawback/late-tip, autopay durable trio, webhook
guards + link last-minute, refunds/invoice lifecycle). Target ≤ ~2.5 min serial, measured. Slow
rate-limiter-bound suites (proposals crud) deliberately excluded from the smoke; the full suite
remains stage 2's job.
