
- [ ] **Step 2: Downscale + encode + write the module via Python Pillow**

Run this exact script. It handles the space in the source path, uses Lanczos resampling, preserves RGBA transparency, and emits the exact module shape `export const LOGO_BASE64 = "data:image/png;base64,…";`:

```bash
python - <<'PY'
import base64, io
from PIL import Image

SRC = r"C:/Users/dalla/Downloads/shopping list/design-system/assets/logo-character-dark.png"
OUT = r"C:/Users/dalla/drb_os/os/client/src/components/ShoppingList/logoBase64.js"

img = Image.open(SRC).convert("RGBA").resize((200, 200), Image.LANCZOS)

buf = io.BytesIO()
img.save(buf, format="PNG", optimize=True)
b64 = base64.b64encode(buf.getvalue()).decode("ascii")

with open(OUT, "w", encoding="utf-8", newline="\n") as f:
    f.write('export const LOGO_BASE64 = "data:image/png;base64,' + b64 + '";\n')

print("png_bytes:", len(buf.getvalue()), "b64_chars:", len(b64))
PY
```

Expected: prints `png_bytes:` roughly 20000–45000 and `b64_chars:` roughly 27000–60000 — well under the 129 KB the old module carried.

- [ ] **Step 3: Verify the written module decodes to a valid 200×200 transparent PNG**

```bash
python - <<'PY'
import re, base64, io
from PIL import Image

txt = open(r"C:/Users/dalla/drb_os/os/client/src/components/ShoppingList/logoBase64.js", encoding="utf-8").read()
m = re.search(r'data:image/png;base64,([A-Za-z0-9+/=]+)', txt)
assert m, "data URI not found in module"
img = Image.open(io.BytesIO(base64.b64decode(m.group(1))))
print("format:", img.format, "size:", img.size, "mode:", img.mode)
assert img.format == "PNG", img.format
assert img.size == (200, 200), img.size
assert img.mode in ("RGBA", "LA", "P"), img.mode
print("OK valid 200x200 PNG with alpha")
PY
```

Expected: `format: PNG size: (200, 200) mode: RGBA` then `OK valid 200x200 PNG with alpha`.
